+ (Object) builder(options = {})

Exposes tuning options for an Undertow web server by returning an options map that includes an Undertow::Builder instance mapped to :configuration. It takes a superset of the options for TorqueBox::Web.run and the map it returns may be passed directly to run. Be sure to pass the desired :host and :port here, so that the proper listener will be created.

The return value is an options map with the :configuration option replacing the ones relevant to an Undertow::Builder

Parameters:

  • options (Hash) (defaults to: {}) —

    a customizable set of options

Options Hash (options):

  • :io_threads (Fixnum) —

    the number of IO threads defaults to available processors

  • :worker_threads (Fixnum) —

    the number of worker threads defaults to 8 * io_threads

  • :buffer_size (Fixnum) —

    in bytes

  • :buffers_per_region (Fixnum) —

    defaults to either 10 or 20 if > 128Mb of RAM

  • :direct_buffers? (Boolean) —

    defaults to true if > 128Mb of RAM

def self.builder(options = {})
  builder = Java::io.undertow.Undertow.builder
  host = options[:host] || DEFAULT_SERVER_OPTIONS[:host]
  port = (options[:port] || DEFAULT_SERVER_OPTIONS[:port]).to_i
  builder.addHttpListener(port, host)
  builder.setIoThreads(options[:io_threads]) if options[:io_threads]
  builder.setWorkerThreads(options[:worker_threads]) if options[:worker_threads]
  builder.setBufferSize(options[:buffer_size]) if options[:buffer_size]
  builder.setBuffersPerRegion(options[:buffers_per_region]) if options[:buffers_per_region]
  builder.setDirectBuffers(options[:direct_buffers?]) unless options[:direct_buffers?].nil?
  result = options.reject { |k, v| TUNING_OPTIONS.include?(k) }
  result[:configuration] = builder
  result
end
